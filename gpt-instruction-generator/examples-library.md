## 2. Example 1: Educational Design Assistant (TCEPFT)  

- **Domain:** Education  
- **Framework:** TCEPFT  
- **Task:** Design inclusive online learning modules for undergraduates.  
- **Context:** Focus on ADA compliance, diverse learning styles, and evidence-backed pedagogy.  
- **Prompt Snippet:**  
  ```
  You are an expert learning experience designer. Create an ADA-compliant online module for undergraduate students, incorporating visual, auditory, and kinesthetic learning styles, grounded in evidence-based pedagogical practices.
  ```
- **Rationale:** Demonstrates comprehensive Task, Context, Persona alignment for complex educational design.

## 3. Example 2: Customer Service Bot (COOPET)  

- **Domain:** Customer Support  
- **Framework:** COOPET  
- **Task:** Provide polite, efficient refund information.  
- **Context:** Customers value empathy and clarity.  
- **Prompt Snippet:**  
  ```
  Context: Handling refund inquiries.  
  Objective: Deliver clear, empathetic, and fast refund responses.  
  Operation: Use company-approved concise templates.  
  Persona: Friendly customer support agent.  
  Example: "Thank you for your request. Your refund will be processed in 3-5 business days."  
  Tone: Warm and professional.
  ```
- **Rationale:** Clear component use supports fast prototyping and consistency.

## 4. Example 3: Technical Documentation Generator (KERNEL)  

- **Domain:** Software Documentation  
- **Framework:** KERNEL  
- **Task:** Generate API docs with detailed error explanations.  
- **Context:** API developers need precise usage and troubleshooting support.  
- **Prompt Snippet:**  
  ```
  Knowledge: API endpoints and error codes.  
  Execution: Write documentation with example requests and responses.  
  Reasoning: Explain common errors and fixes.  
  Exchange: Anticipate user questions.  
  Learning: Tailor explanation depth to user expertise.
  ```
- **Rationale:** Modular structure enables detailed, adaptive documentation.

## 5. Example 4: UX Research Interviewer (CARE)  

- **Domain:** User Experience Research  
- **Framework:** CARE  
- **Task:** Conduct thorough user interviews focusing on accessibility.  
- **Context:** Gathering inclusive feedback for feature redesign.  
- **Prompt Snippet:**  
  ```
  Context: Conducting user interviews on accessibility.  
  Action: Ask open-ended questions probing experience and challenges.  
  Role: UX researcher.  
  Expectation: Collect actionable insights to inform design.
  ```
- **Rationale:** Simple, goal-focused prompt ideal for qualitative insights.

## 6. Example 5: Social Media Content Generator (RTF)  

- **Domain:** Marketing  
- **Framework:** RTF  
- **Task:** Create engaging social media posts.  
- **Context:** Brand voice is casual and upbeat.  
- **Prompt Snippet:**  
  ```
  Role: Social media marketer.  
  Task: Write engaging posts promoting product features.  
  Format: Short, lively text with hashtags.
  ```
- **Rationale:** Minimalist for rapid iteration and informal contexts.

## 7. Example 6: Legal Document Reviewer (TCEPFT)  

- **Domain:** Legal  
- **Framework:** TCEPFT  
- **Task:** Review contract for risk areas and compliance clauses.  
- **Context:** Jurisdiction-specific legal requirements.  
- **Prompt Snippet:**  
  ```
  You are a legal expert reviewing contracts. Identify risks and compliance gaps with jurisdiction X laws. Provide concise summary and recommendations.
  ```
- **Rationale:** Requires detailed context, domain specificity, and clear task framing.

## 8. Summary and Patterns  

Best prompts are explicit about roles, context, and expected outputs. Framework choice depends on task complexity and domain needs. Consistent tone and format improve output reliability.
